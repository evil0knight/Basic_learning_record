## 新增内容时的更新规则

### 在「书中自有黄金屋」新增一本书/学科

需要同步更新以下两个文件：

1. **`书中自有黄金屋/MOC.md`**

   - 在对应分类（计算机科学 / 电子电路 / 其他）下新增一行：
     ```
     - [学科名称](./学科文件夹名/MOC.md)
     ```
2. **`README.md`**

   - 在 `## 📚 书本基础知识` 表格中新增一行：
     ```
     | 学科名 | ⏳ 待学 | [→](./书中自有黄金屋/学科文件夹名/MOC.md) |
     ```

---

### 在「术中自有万钟粟」新增一个技术点

需要同步更新以下两个文件：

1. **`术中自有万钟粟/MOC.md`**

   - 在 `## 技术入口` 表格中新增一行：
     ```
     | 技术名 | 简介 | [技术名 笔记](./技术文件名.md) |
     ```
2. **`README.md`**

   - 在 `## ⚙️ 技术栈` 表格中新增一行：
     ```
     | 技术名 | 学习中 | [→](./术中自有万钟粟/技术文件名.md) |
     ```

---

### 在「库中车马多如簇」新增一个模块

需要同步更新以下两个文件：

1. **`库中车马多如簇/MOC.md`**

   - 在 `## 模块入口` 表格中新增一行：
     ```
     | 模块名 | 简介 | [模块名 笔记](./模块文件名.md) |
     ```
2. **`README.md`**

   - 在 `## 🔩 模块库` 表格中新增一行：
     ```
     | 模块名 | 学习中 | [→](./库中车马多如簇/模块文件名.md) |
     ```

## 上传 GitHub 的操作方式

本项目已通过 SSH 与 GitHub 关联。每次用户说"上传 GitHub"时，
由 Claude 自己依次执行以下命令完成上传：

```bash
git pull
git add .
git commit -m "本次更新了什么"
git push
```

> 说明：commit message 中的"本次更新了什么"应替换为实际更新内容的简短描述。
