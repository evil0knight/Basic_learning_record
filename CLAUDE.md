     ```

## 笔记格式规范

### 书中自有黄金屋 — MOC.md 章节格式

每本书的 MOC.md 中，所有顶层章节的 `<summary>` 标题前必须加 🔴，格式为：

```html
<details>
<summary><h3>🔴章节名</h3></summary>
</details>
```

### 库中车马多如簇 — 模块笔记结构

每个模块建一个子文件夹，内含：

- **`MOC.md`**：模块基础知识（原理、分类、对比等），章节用 `### 🔴标题` 格式，**不使用 `<details>` 折叠**。顶部放笔记导航表格，按类型链接到对应程序文件。
- **程序文件**（每种类型一个 `.md`）：严格遵循 `外设文件模板.md` 规范，供 SKILL 读取。

#### 程序文件规范（SKILL 读取格式）

每个程序文件结构如下：

```
# <模块名>程序

[← 返回 MOC](MOC.md) | [← 主页](../index.md)

---

## <写法名称>（<一句话适用场景>）

<2-3句说明>

​```c
// 完整可复制的代码片段
​```

---

## <写法名称2>（<一句话适用场景>）

...
```

**规则：**
1. 每种写法必须有 `## 标题`，标题即 SKILL 选型菜单里显示的选项名
2. 代码块必须用 ` ```c ` 标注语言
3. 代码块内容要能直接复制使用（含必要的变量声明）
4. 平台相关函数名保留原样，由用户适配

> ⚠️ 禁止在任何标签上使用 `style="display:inline"` 属性，Quartz 构建时会报错。

> ⚠️ 所有 Markdown 链接中的文件扩展名必须小写（用 `.md`，不能用 `.md`），否则 Quartz 部署到 Linux 环境时大小写敏感会导致 404。

> ⚠️ 文件夹名和文件名中不能含有括号（`()`、`（）`），否则 Quartz 渲染后页面中的链接路径会因括号被转义而无法打开。

### Markdown 数学公式

行内公式用 `$...$`，块级公式用 `$$...$$`，不要用 `**\mathbf{...}**`。

## 上传 GitHub 的操作方式

本项目已通过 SSH 与 GitHub 关联。每次用户说"上传 GitHub"时，
由 Claude 自己依次执行以下命令完成上传：

```bash
git pull
git add .
git commit -m "本次更新了什么"
git push
```

> 说明：commit message 中的"本次更新了什么"应替换为实际更新内容的简短描述。
