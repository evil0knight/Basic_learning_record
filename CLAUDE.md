## 新增内容时的更新规则

### 在「书中自有黄金屋」新增一本书/学科

需要同步更新以下两个文件：

1. **`书中自有黄金屋/MOC.md`**

   - 在对应分类（计算机科学 / 电子电路 / 其他）下新增一行：
     ```
     - [学科名称](./学科文件夹名/MOC.md)
     ```
2. **`README.md`**

   - 在 `## 📚 书本基础知识` 表格中新增一行：
     ```
     | 学科名 | ⏳ 待学 | [→](./书中自有黄金屋/学科文件夹名/MOC.md) |
     ```

## 笔记格式规范

### 书中自有黄金屋 — MOC.md 章节格式

每本书的 MOC.md 中，所有顶层章节的 `<summary>` 标题前必须加 🔴，格式为：

```html
<details>
<summary><h3>🔴章节名</h3></summary>
</details>
```

> ⚠️ 禁止在任何标签上使用 `style="display:inline"` 属性，Quartz 构建时会报错。

> ⚠️ 所有 Markdown 链接中的文件扩展名必须小写（用 `.md`，不能用 `.md`），否则 Quartz 部署到 Linux 环境时大小写敏感会导致 404。

> ⚠️ 文件夹名和文件名中不能含有括号（`()`、`（）`），否则 Quartz 渲染后页面中的链接路径会因括号被转义而无法打开。

### Markdown 数学公式

行内公式用 `$...$`，块级公式用 `$$...$$`，不要用 `**\mathbf{...}**`。

## 上传 GitHub 的操作方式

本项目已通过 SSH 与 GitHub 关联。每次用户说"上传 GitHub"时，
由 Claude 自己依次执行以下命令完成上传：

```bash
git pull
git add .
git commit -m "本次更新了什么"
git push
```

> 说明：commit message 中的"本次更新了什么"应替换为实际更新内容的简短描述。
