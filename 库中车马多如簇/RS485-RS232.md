# RS485 / RS232

[← 返回 MOC](MOC.md) | [← 主页](../README.md)

> **一句话**：两种串行通信**物理层标准**，RS232 用于短距点对点，RS485 用于长距多节点工业现场
>
> 注意电平即可

---

## RS232

> 单端信号，全双工，1对1，短距调试常用

### 信号电平

RS232 用**电压高低**表示逻辑，与 TTL 电平**相反且范围更大**：

- **+3V ~ +15V** → 逻辑 **0**（Mark）
- **−3V ~ −15V** → 逻辑 **1**（Space）
- −3V ~ +3V 为过渡区，不确定

MCU 的 UART 引脚是 TTL 电平（0V/3.3V），**接 RS232 设备必须加电平转换芯片**（如 MAX232）。

## RS485 差分信号原理

RS485 用两根线 **A（+）和 B（−）** 传输同一信号的正负两路：

- A − B > +200mV → 逻辑 **1**
- A − B < −200mV → 逻辑 **0**

干扰会同时叠加在 A 和 B 上，差值不变，因此抗干扰能力远强于单端信号（RS232）。

---

## 关键参数

| 参数         | 值                             |
| ------------ | ------------------------------ |
| 最大传输距离 | **1200 m**（9600bps 时） |
| 最大传输速率 | **10 Mbps**（短距离时）  |
| 最多挂载节点 | 32 个（标准），256个（增强型） |
| 工作模式     | 半双工（2线）/ 全双工（4线）   |
| 总线电压范围 | −7V ~ +12V                    |

速率与距离成反比：距离越长，最高速率越低。

---

## 半双工与方向控制

2线 RS485 **同一时刻只能一方发送**，需要用 **DE/RE 引脚** 控制收发方向：

- DE = 1（高）→ 发送使能
- RE = 0（低）→ 接收使能
- 通常将 DE 和 RE 接在同一 GPIO，发送前拉高，发送完立刻拉低

## 与 RS232 对比

| 对比项   | RS232       | RS485        |
| -------- | ----------- | ------------ |
| 信号方式 | 单端        | 差分         |
| 最大距离 | ~15 m       | ~1200 m      |
| 节点数量 | 1对1        | 1主多从      |
| 抗干扰   | 弱          | 强           |
| 常见场合 | 调试/短距离 | 工业现场总线 |
