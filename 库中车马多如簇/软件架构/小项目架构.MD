# 小项目架构

[←MOC](MOC.md)

#### 第 1 层：硬件抽象层 (HAL - Hardware Abstraction Layer)

* **职责：** 直接操作寄存器，完成引脚电平配置、串口波特率初始化、定时器配置，并承载所有原始中断服务程序（ISR）入口。
* **原则：** 不包含任何业务逻辑，仅提供对硬件资源的原子操作接口（如 `GPIO_Set(PB2, HIGH)`）。
* **可移植性：** 更换目标芯片时，仅需重写本层。

#### 第 2 层：设备驱动层 (Driver Layer)

* **职责：** 基于 HAL 层接口，实现具体外设的功能逻辑，如电机超时保护、LED 闪烁时序控制、按键消抖与长短按识别。
* **原则：** 以面向对象风格封装，驱动模块与具体芯片解耦，可在不同硬件平台（如 STM32、国产 MCU）间复用。

#### 第 3 层：应用层 (APP Layer)

* **职责：** 实现业务逻辑，以[表驱法](表驱法.md)状态机为核心，响应系统事件（如 `WIFI_OPEN`）并调用对应驱动接口（如 `Motor_Open()`）。
* **原则：** 不直接访问硬件，仅依赖驱动层接口；代码为纯 C 逻辑，可在宿主机（PC）上独立编译验证。
* **可以单元测试依靠更强大的调试工具**

#### 旁支 (+1 层)：中间件 / 工具层 (Middleware / Utils)

* **职责：** 提供与硬件无关的纯算法模块，如 433MHz 脉冲帧解码、UART 数据帧解析（JSON / 十六进制提取）。
* **特征：** 输入原始数据，输出结构化结果，无任何硬件依赖，可独立单元测试。

### 三、 教你封装：规范的工程目录结构

把上面的架构映射到你的文件夹里，你应该这样建文件夹（你可以把这个当做以后做任何单片机项目的模板）：

**Plaintext**

```
Project_Template/
│
├── 1_App/                 # 应用层：业务逻辑与状态机
│   ├── main.c             # 系统入口：外设初始化，主循环调度
│   ├── app_event.c/.h     # 事件抽象层：定义事件枚举，提供事件发布与订阅接口
│   └── app_state.c/.h     # 核心状态机：表驱法实现
│
├── 2_Middleware/          # 中间件层：与硬件无关的纯算法模块
│   ├── protocol_433.c/.h  # 433MHz 脉宽采样与数据帧组装
│   └── protocol_wifi.c/.h # UART 数据帧解析
│
├── 3_Driver/              # 驱动层：外设功能逻辑封装（面向对象风格）
│   ├── drv_led.c/.h       # LED 闪烁时序控制
│   ├── drv_motor.c/.h     # 电机驱动状态机与超时保护
│   └── drv_key.c/.h       # 按键消抖与长短按识别
│
├── 4_HAL/                 # 硬件抽象层：寄存器操作与中断入口
│   ├── hal_board.c/.h     # 底层初始化汇总（时钟、GPIO）
│   ├── hal_timer.c/.h     # 定时器配置及 ISR 入口
│   ├── hal_uart.c/.h      # 串口配置及接收 ISR 入口
│   └── pin_config.h       # 引脚宏定义（唯一允许出现硬件引脚映射的文件）
│
└── config/                # 全局配置
    └── sys_config.h       # 全局参数宏定义（如 MOTOR_TIMEOUT_MS）
```
