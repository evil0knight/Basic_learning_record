## 1. 广域网 (WAN) 概述

广域网（Wide Area Network, WAN）是指覆盖范围很广（通常跨越城市、国家甚至大洲）的计算机网络。

* **与局域网 (LAN) 的核心区别：**
  * **覆盖范围：** 局域网范围小，广域网范围大。
  * **通信机制：** 局域网通常是**广播/多点接入**网络（如以太网），需要 CSMA/CD 等介质访问控制协议来解决冲突；而广域网的链路主要是**点对点 (Point-to-Point)** 的，即一条线路上只有两端设备， **不存在冲突问题** ，因此不需要 MAC 地址，也不需要 CSMA/CD。
  * **协议层次：** 局域网主要关注物理层和数据链路层；广域网除了这两层，还涉及网络层（如路由器互联）。

## 2. PPP 协议 (Point-to-Point Protocol)

**PPP (点对点协议)** 是目前使用最广泛的数据链路层协议之一。无论你是在家拨号上网（PPPoE），还是两台路由器之间通过专线直连，底层通常都在使用 PPP 协议。

### PPP 协议的三大组成部分

PPP 并不是一个单一的协议，而是一套协议簇。它包含了三个核心组件：

1. **一种将 IP 数据报封装到串行链路的方法：** 规定了帧的格式，支持同步（比特填充）和异步（字符填充）传输。
2. **链路控制协议 (LCP - Link Control Protocol)：** 用于建立、配置、测试和管理数据链路连接。（ *这是物理层连通后，链路层的第一步* ）。
3. **网络控制协议 (NCP - Network Control Protocol)：** 实际上是一组协议。因为 PPP 支持多种网络层协议（不只是 IP），每种网络层协议都有对应的 NCP 来进行协商和配置。对于 IPv4，最常用的是  **IPCP** （IP控制协议），用于为拨号用户动态分配 IP 地址。

## 3. PPP 帧格式

PPP 的帧格式参考了早期的 HDLC 协议，非常简洁。

| 标志 (Flag) | 地址 (Address) | 控制 (Control) | 协议 (Protocol) | 信息部分 (Payload) | 帧检验序列 (FCS) | 标志 (Flag) |
| ----------- | -------------- | -------------- | --------------- | ------------------ | ---------------- | ----------- |
| 1 字节      | 1 字节         | 1 字节         | 2 字节          | 不超过 1500 字节   | 2 字节           | 1 字节      |
| `0x7E`    | `0xFF`       | `0x03`       | *见下文*      | IP数据报或控制报文 | CRC校验          | `0x7E`    |

Export to Sheets

* **标志字段 (Flag - `0x7E`):** 即 `01111110`，表示帧的开始和结束。
* **地址字段 (`0xFF`) 和 控制字段 (`0x03`):** 历史遗留字段，在点对点链路中实际上没有意义，固定为这两个值。
* **协议字段 (2字节):** 非常关键，它告诉接收方“信息部分”装的是什么数据。
  * 如果为 `0x0021`：信息部分是 IP 数据报。
  * 如果为 `0xC021`：信息部分是 LCP 控制报文。
  * 如果为 `0x8021`：信息部分是 NCP (IPCP) 控制报文。
  * 如果为 `0xC023` / `0xC223`：信息部分是身份认证报文 (PAP / CHAP)。

*(注：因为帧首尾标志是 `0x7E`，为了实现透明传输，PPP 协议在异步传输时采用 **字符填充法** ，在同步传输时采用 **零比特填充法** ，这正是你之前学过的组帧技术。)*

## 4. PPP 的工作状态 (状态转移)

建立一个 PPP 连接并不是一蹴而就的，它需要经过严密的“握手”和状态流转：

1. **链路静止 (Link Dead):** 物理层未连接，此时没有载波信号。
2. **链路建立 (Link Establish):** 物理层连接（如拨号成功）后，双方进入此阶段，开始通过 **LCP** 协议协商链路参数（如最大帧长、是否需要身份验证等）。
3. **鉴别 (Authenticate):** (可选阶段) 如果 LCP 协商要求身份验证，则进入此阶段。常用的鉴别协议有  **PAP** （明文密码，两次握手）和  **CHAP** （挑战握手，更安全，三次握手）。如果鉴别失败，直接跳到终止状态。
4. **网络层协议 (Network):** 鉴别成功后进入此阶段。双方通过 **NCP** (如 IPCP) 协商网络层参数。例如，ISP 在这个阶段为你分配一个动态公网 IP 地址。
5. **链路打开 (Open):** 协商全部完成，此时链路就可以互相传输网络层的 IP 数据报了。
6. **链路终止 (Terminate):** 当任何一端发出终止请求（LCP 报文），或者物理链路断开时，进入此阶段，随后回到静止状态。
