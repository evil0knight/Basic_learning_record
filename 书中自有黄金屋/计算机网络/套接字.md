> **一句话**: 套接字就是应用程序委托协议栈收发数据时使用的接口

## 先记两个识别方式

- **描述符**: 应用程序在**本机内部**识别套接字的编号
- **IP地址 + 端口号**: 客户端和服务器在**网络中**识别对方套接字的方式

简单记:

- 描述符: 给自己看
- IP地址和端口号: 给通信对方看

为什么不能只靠IP地址:

- IP地址只能定位到一台主机
- 连接的对象是主机里的**某个具体套接字**
- 所以还必须加上端口号

为什么不能把描述符告诉对方:

- 描述符只在本机内部有效
- 对方并不知道这个描述符

## 客户端收发数据流程

`gethostbyname -> socket -> connect -> write -> read -> close`

### 1. 创建套接字: `socket`

- 协议栈创建套接字,并返回一个**描述符**
- 应用程序把描述符保存起来,以后靠它指定要操作哪个套接字
- 一台计算机可以同时有多个套接字,所以需要描述符来区分

### 2. 连接: `connect`

`connect(描述符, 服务器IP地址, 服务器端口号, ...)`

- 描述符: 告诉协议栈用哪个套接字去连接
- 服务器IP地址: 告诉协议栈连接哪台主机
- 服务器端口号: 告诉协议栈连接主机上的哪个应用

补充:

- 客户端端口号通常由操作系统自动分配
- 服务器端口号通常是事先规定好的,例如 HTTP `80`, SMTP `25`

### 3. 通信: `write` / `read`

- `write(描述符, 发送数据, 数据长度)`: 把数据交给协议栈发送
- 连接建立后,套接字中已经保存了对方的IP地址和端口号
- 所以应用程序只要给出描述符,协议栈就知道把数据发给谁
- `read(描述符, 接收缓冲区, ...)`: 把收到的数据放入接收缓冲区

### 4. 断开: `close`

- 收发结束后调用 `close`
- 连接断开,套接字被删除
- 这样一次通信就结束了

---

## 本章小结

- 描述符: 应用程序在本机内部识别套接字的机制
- IP地址和端口号: 客户端和服务器之间识别对方套接字的机制
- 客户端基本流程: `socket -> connect -> write -> read -> close`
