## 写法一：标题固定，内容可折叠

适合知识点多、想先看标题再按需展开的情况。

#### 总线

<details>
<summary>展开笔记</summary>

总线是连接多个部件的信息传输线，分为**数据总线、地址总线、控制总线**。

- 数据总线：双向，宽度决定一次传输的数据量
- 地址总线：单向，宽度决定寻址范围
- 控制总线：传输控制信号，方向不定

```
寻址范围 = 2^地址总线位数
```

</details>

#### Cache

<details>
<summary>展开笔记</summary>

Cache 利用**程序局部性原理**，分为时间局部性和空间局部性。

命中率 h，Cache 存取时间 $t_c$，主存存取时间 $t_m$：

$$
t_{avg} = h \cdot t_c + (1-h) \cdot t_m
$$

**替换算法：**

| 算法 | 说明                 |
| ---- | -------------------- |
| LRU  | 替换最久未使用的块   |
| FIFO | 替换最早调入的块     |
| LFU  | 替换使用次数最少的块 |

</details>

## 写法二：整个章节折叠

适合已经掌握、平时不需要看的章节。

<details>
<summary><strong>第五章 · 指令系统</strong>（点击展开）</summary>

#### 指令格式

指令 = 操作码 + 地址码，操作码决定操作类型，地址码指出操作数位置。

#### 寻址方式

- **立即寻址**：操作数直接在指令中，速度最快
- **直接寻址**：地址码就是有效地址
- **间接寻址**：地址码指向的单元里存的才是有效地址
- **寄存器寻址**：操作数在寄存器中，不访问内存

</details>

## 写法三：默认展开

加 `open` 属性，默认就是展开状态，可以手动收起。

<details open>
<summary>流水线（默认展开）</summary>

将指令执行分为**取指、译码、执行、写回**等阶段并行处理。

吞吐率 $TP = \dfrac{n}{(k+n-1)\Delta t}$，其中 k 为流水线级数，n 为指令数。

</details>
